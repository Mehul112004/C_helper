# Technical Indicators Logic

All technical indicator computations are housed within `IndicatorService` (`app/core/indicators.py`). It uses `pandas` for highly optimized, vectorized mathematical operations instead of line-by-line loops.

## Core Properties
*   **Caching:** The class maintains a class-level dictionary `_cache` mapping `(symbol, timeframe, latest_candle_time)` to the computed `IndicatorResult`. A cache miss triggers recalculation, while a hit returns instantly. Since the exact timestamp of the newest candle is in the key, it inherently invalidates when real-time updates arrive.

## Core Computation Methods

### `compute_ema(closes, period)`
*   **Description:** Calculates the Exponential Moving Average.
*   **How it Works:** Employs the Pandas `ewm` (Exponentially Weighted Window) function with `adjust=False` to strictly match standard charting platform implementations. It places more weight on recent candles to track trend momentum.
*   **Input:** `closes` (Pandas Series), `period` (Integer).
*   **Output:** Pandas Series representing the EMA values.

### `compute_rsi(closes, period=14)`
*   **Description:** Calculates the Relative Strength Index.
*   **How it Works:** Calculates candle-to-candle price differences to isolate "gains" and "losses". Crucially, instead of simple moving averages, it uses **Wilder's Smoothing method** (which is mathematically identical to an EMA where alpha `1/period`) to calculate average gains/losses. This ensures the output matches Binance and TradingView exactly.
*   **Input:** `closes` (Pandas Series), `period` (Integer).
*   **Output:** Pandas Series representing RSI 0-100 values.

### `compute_macd(closes, fast=12, slow=26, signal=9)`
*   **Description:** Calculates the Moving Average Convergence Divergence.
*   **How it Works:** Computes the `fast` EMA and `slow` EMA, and derives the MACD Line by subtracting the slow from the fast. It then computes the Signal Line by taking an EMA of the MACD Line itself. Finally, the Histogram is generated by subtracting the Signal Line from the MACD Line.
*   **Input:** `closes` (Pandas Series), and integer parameters.
*   **Output:** Dictionary with three Pandas Series keys: `macd_line`, `macd_signal`, `macd_histogram`.

### `compute_bollinger(closes, period=20, std_dev=2.0)`
*   **Description:** Calculates Bollinger Bands.
*   **How it Works:** Uses a pandas `.rolling()` window to calculate both the Simple Moving Average (the middle band) and the standard deviation over the window. The upper/lower bands are derived by adding/subtracting `std_dev * standard_deviation` from the mean. It also calculates normalized `bb_width` which is useful for detecting "squeezes".
*   **Input:** `closes` (Pandas Series), parameters.
*   **Output:** Dictionary with four Pandas Series: `bb_upper`, `bb_middle`, `bb_lower`, `bb_width`.

### `compute_atr(highs, lows, closes, period=14)`
*   **Description:** Calculates Average True Range (usually used for volatility and stop-loss placement).
*   **How it Works:** Calculates the "True Range" of each candle by finding the absolute maximum of three values: 
    1) Current High - Current Low
    2) Current High - Previous Close
    3) Current Low - Previous Close.  
    It then rolls a simple moving average over this True Range.
*   **Input:** Three Pandas Series (`highs`, `lows`, `closes`), period.
*   **Output:** Pandas Series representing ATR values.

## Main Entry Point

### `compute_all(symbol, timeframe, include_series)`
*   **Description:** Main service coordinator used by blueprints.
*   **How it Works:**
    1. Fetches a slice of `MIN_CANDLES_IDEAL` (~250) recent candles using SQLAlchemy.
    2. Constructs a cache key. If a match is found, retrieves data from RAM.
    3. If cache misses, transforms the SQL result into a pandas DataFrame and invokes all static indicator functions above simultaneously.
    4. Extracts the `.iloc[-1]` (latest) value from every series to construct the `latest` summary response.
    5. Cleans and aligns NaN values for JSON compatibility, caching the result alongside a list of `warnings` if the candle dataset was too small.
*   **Input:** `symbol` (str), `timeframe` (str), `include_series` (bool).
*   **Output:** Comprehensive dictionary of calculations, ready to be serialized to JSON.
